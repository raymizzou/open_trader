# 趋势交易系统复盘设计

## 状态

- 日期：2026-07-16
- 状态：已批准
- 范围：东方财富 A 股、老虎美股、辉立港股的趋势策略复盘

## 目标

为每个趋势策略市场建立独立复盘页，回答三个问题：

1. 当前执行的策略和版本是什么，完整参数是什么。
2. 纪律模拟、用户实际执行和本市场基准的表现分别如何。
3. 发生代码或查询缺陷后，能否基于冻结输入重新计算而不改写原始事实。

本期不实现参数编辑或回测入口。未来能力应为“导出参数到回测”：复制当前版本
的完整参数为可编辑回测配置，不能直接修改运行版本。

## 账户内入口

复盘入口位于各券商账户头部的 `当天趋势报告` 按钮旁，不增加全局入口：

| 券商账户 | 市场 | 入口文字 | 市场基准 |
|---|---|---|---|
| 东方财富 | A 股 | `A 股复盘` | 中证全指（前复权） |
| 老虎 | 美股 | `美股复盘` | SPY（前复权） |
| 辉立 | 港股 | `港股复盘` | 恒生综合指数（前复权） |

富途账户只显示跨市场期权关注，不显示趋势复盘入口。入口按钮复用现有暖色描边按钮样式，不使用
黑色主按钮。点击后只打开该账户对应市场的复盘，A 股数据不得与港股或美股合并。

## 复盘对象与周期

每个市场独立维护一套策略、版本、纪律模拟账户、实际执行记录和市场基准。首个
复盘批次从模拟实验以全现金、零持仓启动之日开始，到该市场完成第 30 笔纪律模拟
交易为止；后续按不重叠的每 30 笔完整交易形成新批次。

一个批次内的纪律模拟、实际执行和市场基准使用完全相同的起止交易日。交易样本
跨版本时按交易入场时的版本归属，不追溯改写。

## 策略身份与完整参数

复盘页顶部直接显示：

- 策略名称。
- 策略版本。
- 市场和券商账户。
- 当前运行代码实际采用的完整参数。

报告生成时冻结机器可读的 `strategy_snapshot`。它包含 `strategy_id`、
`strategy_name`、`strategy_version`、`market`、`effective_from`、生成进程的
`process_version`，以及下表列出的全部 `parameters`，不能只保存参数摘要。

Dashboard 参数表只渲染该快照，不在前端复制策略常量，也不从说明文字推断参数。
策略生成、模拟执行、复盘和未来回测导出必须消费同一个参数快照。

当前 A 股运行代码的参数表必须完整包含以下内容：

| 类别 | 参数 | 当前值 |
|---|---|---|
| 候选来源 | 趋势动物组合 | 温转热（A 股）和温转热（ETF 基金个股） |
| 入场过滤 | 交易市场 | 沪深 A 股；排除北交所、ST、`*ST` 和退市标记 |
| 入场过滤 | 趋势温度 | 前一状态为温；当前状态为热或沸 |

| 入场过滤 | 趋势强度 | 不低于 95 |

| 入场过滤 | 趋势节气 | 谷雨、立夏或夏至 |
| 入场过滤 | 总市值 | 不低于 100 亿元 |

A 股参数。

实现时必须先以已部署代码生成的快照为运行事实，并同步修正文档；复盘页不得展示
已经失效的旧门槛。

## 三条对比曲线

### 纪律模拟

富途为港股、美股和 A 股提供独立模拟账户。每个市场模拟账户只运行本趋势实验，
因此直接使用整个模拟账户，不增加虚拟子账户。

- 从全现金、零持仓开始。
- 每个正式纪律动作在有效窗口自动提交模拟盘。
- 买入和卖出统一使用市价单；港股和 A 股从模拟盘支持的连续交易时段开始执行。
- 下单数量按模拟账户当时净值和策略目标仓位重新计算，不复制真实账户绝对股数。
- 纪律曲线直接采用富途模拟账户日终净值；不在本地重复估算或扣除费用。
- 每日冻结订单状态、持仓、现金和收盘净值。

### 实际执行

实际执行曲线使用用户真实执行并登记的同市场、同策略成交及每日收盘持仓价值，
计入真实费用。未登记、无法归属该策略或无法取得日终净值的数据不得猜测补齐；
对应指标显示为数据不足。

纪律外交易保留在实际执行曲线中，因为该曲线衡量用户实际得到的结果；纪律外交易
不得进入纪律模拟样本、策略调参样本或 Kelly 样本。

### 市场基准

基准由 Futu 行情接口按日冻结前复权收盘价：A 股为中证全指 `SH.000985`，美股为
SPY `US.SPY`，港股为恒生综合指数 `HK.800701`。基准在复盘批次首日按与策略相同
的初始净值归一化，使用与两条策略曲线相同的交易日区间，不依赖人工维护的文件。

## 五项指标

主页面只展示以下五项，不增加 Alpha、Beta、胜率、盈亏比、Sortino 或综合结论：

1. **期间净收益率**：`期末净值 / 期初净值 - 1`，已扣全部费用。
2. **相对市场超额收益**：策略期间净收益率减市场基准期间净收益率；基准自身为 0。
3. **最大回撤**：每日净值相对此前峰值的最大跌幅。
4. **卡玛比率**：几何年化收益率除以最大回撤绝对值。
5. **夏普比率**：以本市场交易日日收益计算的年化超额收益均值除以年化波动率。

卡玛和夏普使用同一批次、同一交易日历和冻结的无风险收益率参数。样本不足、最大
回撤为零或收益波动为零时不生成虚假数值，明确显示数据不足。

每项同时展示三组值：

- 纪律模拟。
- 实际执行。
- 市场基准。

## Dashboard 展示

复盘页只包含以下内容，顺序固定：

1. 券商、市场、策略名称和策略版本。
2. 当前版本完整参数表。
3. `收益与回撤` 分组条形图：期间净收益率、相对市场超额收益、最大回撤。
4. `风险调整收益` 分组条形图：卡玛比率、夏普比率。

条形图直接标注数值，并用颜色和文字图例同时区分纪律模拟、实际执行和市场基准。
375px 移动视口不得横向滚动；参数表在窄屏转换为纵向参数行，不截断参数值。

本期明确不显示：

- 复盘结论卡。
- 运行状态卡。
- 回测步骤或版本流程图。
- 参数编辑、导出或回测按钮。
- 缺陷回放入口。
- 五项指标之外的其他统计卡。

## 缺陷回溯

原始报告、模拟成交、实际成交和原始净值曲线都是不可变事实。发现代码、参数映射
或 query 缺陷后：

1. 先锁定受影响的日期、报告、标的、交易和代码版本。
2. 使用当时冻结的趋势动物响应、查询参数、行情、账户和策略参数快照重放。
3. 新增纠正回放产物，记录原产物、缺陷原因、重放时间和修复 Git SHA。
4. 不覆盖原始报告、成交或原始曲线。

如果原 query 没有取得必要字段，且趋势动物不再提供对应历史日期，回放结果必须标记
为缺少历史输入，不能补猜。缺陷回放数据本期不在主复盘页展示，但冻结输入和纠正
产物必须可供审计。

## 数据产物

每个市场、每个 30 笔批次保存一个不可变 JSON 产物，至少包含：

- 批次编号和起止日期。
- 券商、市场、账户和基准身份。
- 完整 `strategy_snapshot`。
- 纪律模拟、实际执行和市场基准的每日净值。
- 无风险收益率参数和逐日冻结的基准事实。
- 五项指标及数据不足原因。
- 30 笔纪律模拟交易的报告、订单和成交引用。
- 生成时间和 Git SHA。

Dashboard API 只投影该产物，不在请求时重新计算策略参数或历史指标。

## 错误处理

- 模拟账户不可用：不更新纪律模拟曲线，记录中断，不伪造成现金持有。
- 实际成交或日终净值缺失：实际执行指标显示数据不足，不使用模拟值替代。
- 基准缺失：该批次五项对比指标不可发布，保留已有完整产物。
- 参数快照与报告动作使用的参数不一致：报告生成失败，不展示错误版本。
- 单日重复写入：按市场、账户、交易日和版本去重，保持原子写入。
- 指标计算出现非有限值：保存具体原因，不输出 `NaN` 或无穷大。

## 验证与验收

自动化验证至少覆盖：

- 三个账户只显示各自市场的复盘入口，老虎不显示。
- 参数表与冻结 `strategy_snapshot` 完全一致，并覆盖 A 股所有当前参数。
- 三条净值曲线使用相同日期和初始净值。
- 纪律模拟使用富途权威日终净值，不叠加本地费用估算。
- 五项指标公式、边界和数据不足状态。
- 条形图只出现五项指标和三组对比值。
- 页面没有复盘结论、状态卡、回测入口或其他未批准内容。
- 375px 页面无横向滚动，完整参数可读。
- 缺陷回放创建新产物且不修改原始事实。

实际验证遵循项目 Dashboard 验收门：

1. 运行相关单元测试并记录精确结果。
2. 使用真实模拟账户、实际账户数据和真实市场基准生成一次复盘批次。
3. 检查运行进程、Git SHA、PID 和新日志，重启仍加载旧代码的进程。
4. 在最后一步运行 `make acceptance`；只有 `PASS` 才可请求用户验收。
5. 重新部署完全相同的已验收 Git SHA，核对新 PID、工作目录、Git SHA、新日志和
   Review URL 的 HTTP 200。
