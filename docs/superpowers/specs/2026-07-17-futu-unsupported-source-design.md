# 富途不支持异动字段的状态设计

## 目标

富途原生异动接口明确返回 `err_code=-12301` 时，将对应字段标记为“不适用”，避免把确定的产品能力限制误报为数据故障。连接失败、响应异常、字段缺失和过期数据仍然阻断决策。

## 方案

采用独立状态 `not_applicable`：

- 仅 `-12301` 映射为 `not_applicable`，保留标的和模块原因。
- `futu_skill_facts` 记录可同时包含正常模块和不适用模块；不适用模块不计入生成失败。
- 决策来源完整性检查接受 `ok` 与 `not_applicable`，继续拒绝 `missing`、`error`、`stale_run_date` 等状态。
- Dashboard 展示“不适用”，不伪造信号、摘要或替代指标。

不采用以下方案：

- 忽略全部技术异动错误：会掩盖真实故障。
- 用 K 线自行计算替代：改变了数据来源，不属于本次修复。

## 数据流

`Futu SDK -12301` → 异动客户端的专用“不支持”异常 → 抽取记录 `status=not_applicable` → 来源完整性检查放行 → Dashboard 显示“不适用”。

其他异常仍沿用现有 `status=error` 流程。

## 验证

- 单元测试证明 `-12301` 生成 `not_applicable`，且不计入失败。
- 单元测试证明完整性检查放行 `not_applicable`，仍阻断其他非正常状态。
- 重新抽取真实港股富途字段并确认 `00200/02623/02824` 为“不适用”。
- 运行完整测试、真实 Dashboard 工作流和最终 `make acceptance`。
