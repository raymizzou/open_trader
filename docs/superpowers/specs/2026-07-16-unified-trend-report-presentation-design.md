# 趋势报告表现层统一设计

## 状态

- 日期：2026-07-16
- 状态：已批准
- 范围：Dashboard 中 A 股、美股、港股当天趋势报告的表现层

## 目标

以现有 A 股“动作优先”报告为唯一视觉骨架，统一三类市场报告的信息层级、
区块顺序、表格、空状态、审计区和移动端布局。交易规则、报告生成逻辑和报告
数据不变。

## 已选方案

复用现有 A 股报告工作区和样式，只在市场确有差异时切换表格字段与交易时段。
不复制一套美股/港股页面，不用 CSS 把旧卡片伪装成表格，也不引入新的组件层、
依赖或设计系统。

## 统一结构

三类市场按同一顺序展示：

1. 券商、市场、报告日期、数据日期、生成时间和账户状态。
2. 正式买入、全部卖出、继续持有、人工复核四个计数。
3. `优先处理 · 卖出触发`。
4. `需要确认 · 人工复核`。
5. 对应市场交易时段的 `正式买入计划`。
6. `盘中持续 · 已有持仓`。
7. `审计详情`。

非空动作区使用同款表格；空动作区显示同款“无”状态。标题、字号、间距、
边框、圆角、背景与动作状态色统一复用 A 股样式。颜色不是识别动作的唯一方式。

## 市场字段边界

A 股保留温度、趋势节气、行业温度等已有字段。美股和港股仅展示报告中已有的
执行参考价、强度、原因、金额上限、预计股数和保护线等事实。市场没有的字段
不创建空列；单条记录缺失已有字段时显示 `—`，不得推导或伪造 A 股字段。

审计区外观统一，内容继续使用各市场现有审计数据和排除原因。美股/港股账户
不参与项继续保留在审计详情中。

## 响应式布局

桌面端使用统一表格密度。760px 以下沿用现有 A 股表格转事实卡规则，动作、标的、
原因与保护线优先展示；375px 视口不得产生页面级横向滚动，交互目标至少 44px 高。

## 数据完整性

表现层不得改变统计或筛选结果。计数必须与动作明细一致。当前美股验收报告必须
继续显示买入 `EA`、`PYPL`，人工复核 `BOTZ`、`RAM`；仅呈现方式改变。

## 非目标

- 不修改交易纪律、候选筛选、仓位计算或保护线计算。
- 不重生成或改写趋势报告 JSON/Markdown。
- 不修改持仓主表、券商同步或通知流程。
- 不新增通用渲染配置、前端框架或依赖。

## 验收条件

1. A 股、美股、港股使用相同工作区骨架和动作区顺序。
2. 三类市场非空区均为统一表格，空区均显示统一“无”状态。
3. 市场专属字段正确保留，缺失字段不被伪造。
4. 页面计数与明细一致；当前美股报告保留 2 个买入和 2 个人工复核标的。
5. 审计区样式统一且原有市场审计内容未丢失。
6. 桌面端无异常溢出；移动端转为统一事实卡且 375px 无横向滚动。
7. 返回持仓及报告入口可点击并正确跳转。
8. 自动化检查覆盖三类市场、真实报告数据和桌面/移动浏览器流程。
9. 最终 `make acceptance` 必须返回 `PASS`。
10. 验收后重新部署完全相同的 Git SHA，并验证 PID、工作目录、Git SHA、新日志和
    review URL HTTP 200。
