# 按账户分组的持仓与策略 Dashboard 设计

日期：2026-07-14

## 目标

把独立的老虎长线策略面板并入现有持仓列表，使页面以“券商账户”为第一层级，同时保留全组合总览、市场筛选和现有标的详情。

页面必须让用户快速回答三个问题：资产在哪个账户、该账户承担什么交易周期、每个持仓受什么策略约束。

## 账户定位

| 券商 | 周期标签 | 策略名称 | 第一版策略状态 |
| --- | --- | --- | --- |
| 富途 | 中短线 | 股票与期权 | 指标待接入 |
| 老虎 | 长线 | SMA200 组合策略 | 显示真实组合指标与逐标的状态 |
| 辉立 | 中线 | 中线策略 | 指标待接入 |
| 东方财富 | 偏短线 | 趋势交易 | 指标待接入 |

第一版不得把现有单标的交易计划伪装成富途、辉立或东方财富的账户级组合策略。

## 页面结构

1. 顶部保留全组合总资产、市场筛选和四个账户汇总卡。
2. 删除重复的券商筛选按钮。账户卡承担账户导航职责，不再承担过滤职责。
3. 删除独立的“老虎长线组合”面板。
4. 主内容改名为“持仓与策略”，依次显示富途、老虎、辉立、东方财富四个账户区块。
5. 四个账户默认全部展开，不提供折叠状态或折叠控制。
6. 顶部账户卡使用原生页内链接定位到对应账户区块；不隐藏其他账户。

## 账户区块

每个区块由三部分组成：

### 账户头

显示券商名称、脱敏账户别名、周期标签、账户总资产、持仓资产、现金、持仓数量、数据来源和数据时间。

### 策略摘要

- 老虎：显示 SMA200 策略的年化收益、最大回撤、夏普比率、卡玛比率和中文门槛状态。
- 其他账户：显示周期和策略名称，并明确标注“策略指标待接入”。
- 单个策略产物缺失只影响策略摘要，不隐藏或阻断该账户持仓。
- 所有内容只供人工复核，不增加下单入口。

### 券商颜色

- 只给账户头和策略摘要使用低饱和浅色背景，持仓表保持白底。
- 富途使用浅蓝，老虎使用浅橙，辉立使用浅绿，东方财富使用浅红。
- 桌面和手机使用相同映射，不增加渐变、动画或新的交互状态。
- 券商名称、周期和策略文字始终可见；颜色只辅助区分，不作为唯一识别方式。

### 持仓表

- 按账户展示原始券商持仓；同一标的位于不同券商时必须分别显示。
- 不再在账户内部增加市场分组标题，市场作为普通列显示。
- 保留现有数量、成本价、实时价、市值、盈亏和详情入口。
- “账户权重”和“组合权重”使用两个独立数字列，避免在一个单元格内混排。
- 持仓表不显示逐行“策略”列；策略信息只保留在账户顶部策略摘要中。

## 筛选与详情

- `全部市场 / US / HK / A 股` 继续同时作用于四个账户内的持仓。
- 市场筛选后仍显示四个账户头；无匹配持仓的账户显示中文空状态。
- `现金` 保留现有现金明细视图。
- 点击持仓继续打开现有标的详情；研究、交易计划和深链接行为不变。
- 筛选、刷新或关闭详情后，不保存额外的账户展开状态，因为所有账户始终展开。

## 数据流

1. 继续使用现有 `broker_summaries`、`source_statuses`、`holdings[].broker_details` 和现金数据。
2. 前端从合并持仓的 `broker_details` 还原逐券商行；只有缺少明细且持仓仅属于一个券商时才使用合并行回退。
3. 使用现有券商汇总计算区块资产，并用账户总资产计算账户内占比。
4. 只有老虎区块读取现有 `tiger_long_term_strategy`；按标准化标的代码关联策略成员。
5. 账户周期与策略名称是固定的 Dashboard 展示配置；第一版不增加编辑界面或新配置文件。

## 桌面与手机

### 桌面

- 账户头、策略摘要、持仓表形成一个连续卡片。
- 账户权重和组合权重各自右对齐，使用清晰的独立表头。
- 数字列使用等宽数字并保持右对齐。

### 手机

- 每个账户保持独立区块，账户头垂直排列。
- 每个持仓保持两行卡片：第一行显示标的、市值、账户权重、组合权重和盈亏；第二行显示市场、数量、价格和详情操作。
- 次要字段通过现有详情入口查看。
- 不允许页面或持仓区域出现横向滚动。

### 可访问性

- 账户卡使用可键盘操作的原生链接，并保留清晰的焦点状态。
- 账户区块使用标题和 `aria-labelledby` 建立语义关系。
- 周期和策略状态必须使用文字表达，不能只依赖颜色。
- 保留现有 `prefers-reduced-motion` 支持；页内定位不增加强制动画。

## 错误与空状态

- 账户没有持仓：显示“当前筛选下没有持仓”。
- 策略未接入：显示“策略指标待接入”。
- 老虎策略产物缺失或无效：显示现有错误原因，持仓仍正常显示。
- 券商资产明细不完整：沿用现有不可用标记，不用零值代替未知值。
- 不在页面显示英文内部状态代码。

## Acceptance

每次修改后必须运行 `make acceptance`，且只有 `PASS` 可以交付。门禁新增以下浏览器断言，并同时覆盖桌面与手机：

1. 四个账户区块存在并全部展开。
2. 四个周期与策略标签正确。
3. 页面不存在独立的 `tiger-long-term-panel`。
4. 老虎策略指标和老虎持仓位于同一账户区块。
5. 老虎账户顶部显示中文策略指标；其他账户顶部显示“策略指标待接入”，持仓表不显示策略列。
6. 点击账户卡会定位到对应区块，且其他区块仍可见。
7. 市场筛选对四个账户生效。
8. 手机页面无横向滚动，持仓卡包含核心字段。
9. 缺少策略产物不影响持仓展示。
10. 页面不包含下单入口，也不泄漏英文内部状态代码。

验收通过后，必须按项目规则重启到同一个已验收 Git SHA，并核对新 PID、工作目录、Git SHA、新日志和 HTTP 200。

## 明确不做

- 不实现富途、辉立或东方财富的新账户级策略。
- 不汇总单标的信号为账户回测指标。
- 不增加账户折叠、拖拽排序、用户自定义标签或策略配置编辑器。
- 不改变现有策略产物、回测逻辑或下单边界。
