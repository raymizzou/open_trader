# 飞书 A 股趋势报告中文展示设计

## 目标

把飞书报告从“机器审计文本”改成“人工执行清单”。交易数据、策略判断、仓位和 API 调用保持不变；只调整 Markdown/飞书展示层。

## 默认层级

飞书与本地 Markdown 采用单列“操作优先”结构：

1. 日期、账户新鲜度和卖出/买入/持有数量摘要。
2. 全部卖出清单：标的、中文原因、活动保护线。
3. 买入清单：固定顺序、有效时间、估算股数、金额上限、预计保护线。
4. 继续持有与人工复核清单。
5. 中文附录：候选榜、排除理由、行业集中度、数据来源和 API 成本。
6. 免责声明。

## 中文化边界

Markdown 和飞书不得展示内部英文枚举或字段名。使用一个集中式、显式映射表翻译：

- 动作：`SELL_ALL` → `全部卖出`，`HOLD` → `继续持有`，`MANUAL_REVIEW` → `人工复核`。
- 持仓原因：例如 `left_trend_right_side` → `右侧趋势已结束`，`danger_signal` → `危险信号触发`，`trend_intact` → `趋势保持完好`。
- 排除原因：例如 `right_side_days_not_below_10` → `进入右侧趋势已满 10 天`，`amount_below_1` → `日成交额不足 1 亿元`，`atr_unavailable` → `缺少 ATR 数据`。
- API 事实只展示中文摘要，不展示 `rows=`、字段列表、`cache=client-managed` 等实现细节。
- 本地绝对路径显示为业务名称，例如“东方财富账户快照”。

未知内部代码必须以“未知原因（原代码）”显示，不能静默丢失，也不能影响报告生成。

## 审计数据

JSON 继续保存完整原始字段、内部动作/原因代码、候选事实、缓存命中、成本和进程版本。此次改动不修改 JSON schema，也不删除审计信息。

## 数据流

既有 `TrendReport` 仍是唯一输入。Markdown 渲染器先调用中文标签映射，再按操作优先层级输出；飞书继续发送冻结后的同一份 Markdown。JSON 渲染路径不经过中文标签映射。

## 错误与兼容

- 缺少可选数据时显示“未知”或省略对应细节，不编造数值。
- 没有正式动作时继续显示既有“无正式操作”语义。
- 飞书发送、冻结报告、receipt 恢复和 revision 行为不变。
- 历史冻结报告不重写、不补发；新格式从下一份报告开始生效。

## 验证

最小测试覆盖：

- 动作、持仓原因和排除原因均为中文。
- 主操作区不出现已知内部英文代码。
- API 技术细节只在 JSON 中保留。
- 卖出、买入、持有、无动作和未知原因均可稳定渲染。
- 飞书发送内容与冻结 Markdown 完全一致。

## 非目标

- 不改策略纪律、候选排序、仓位计算、保护线、API 或调度。
- 不引入飞书卡片协议、模板引擎或新依赖；现有 Markdown 文本足够。
