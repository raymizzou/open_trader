# 模拟盘持仓与趋势报告历史设计

## 目标

在每个趋势交易账户内，把真实持仓、模拟盘持仓、趋势报告和复盘变成四个同级视图。模拟盘持仓必须来自对应富途模拟账户，并能追溯到触发入场的精确报告版本；新报告生成后，旧报告及执行结果仍可查询。

## 范围

账户内固定显示以下顺序的文字 Tab：

1. 真实持仓
2. 模拟盘持仓
3. 趋势报告
4. 美股复盘、港股复盘或 A股复盘

真实持仓默认打开。切换 Tab 只替换账户下方的内容，不改变账户头部，不增加账户卡片或新的全局导航。

“历史报告”位于“趋势报告”页面内，不是第五个账户级 Tab。

本期不做模拟盘历史仓位曲线、每日仓位快照、数据库迁移、手工关联、参数导出或回测入口。

本期的趋势交易账户仅指老虎美股、辉立港股和东方财富 A 股。富途真实账户现有的期权关注入口不改成这组 Tab。

## 依赖

模拟持仓与报告的可靠关联依赖 `fix/trend-review-execution-closure` 中的稳定动作标识、富途历史订单查询和不可变执行事件。实现前该能力必须已经合入 `main`，或由用户明确批准集成对应提交；不能重新实现第二套执行账本。

## 界面设计

### 账户头部

沿用已确认的两层布局：第一行只放账户身份、资产事实和总资产；第二行放四个低调的文字 Tab。Tab 使用现有颜色、字号、间距和焦点样式，以文字粗细和底边线表达当前状态，不使用描边按钮、胶囊或新卡片。

### 真实持仓

保留现有真实持仓表格、数据字段和交互，作为默认视图。除被 Tab 容器包裹外，不改变其内容和视觉结构。

### 模拟盘持仓

使用与真实持仓相同的表格结构和格式。市场、代码、名称、数量、成本、实时价、市值、权重和盈亏均来自对应富途模拟账户与现有行情换算路径。

对于系统可追溯的仓位，在标的名称的次要信息位置显示可点击的 `报告 YYYY-MM-DD · 版本`。点击后打开该仓位入场所依据的精确冻结报告，不跳到当前最新报告。

系统启用前或无法可靠关联的模拟仓位仍然展示，并标记“未关联历史报告”。不能隐藏或猜测其来源。

当前没有模拟持仓时，表格区域显示“当前无模拟盘持仓”，不使用示例行填充。

### 趋势报告

默认显示当前有效趋势报告。页面内提供低调的“历史报告”入口，进入后按报告执行日期倒序排列，并在同一日期下列出所有版本。

历史条目至少显示执行日期、数据截止日期、生成时间、策略版本和执行摘要。点击条目打开只读的精确冻结报告及对应执行结果。返回后保留原账户、Tab 和滚动位置。

### 复盘

保持现有复盘内容与指标展示。本期只把入口纳入同级 Tab，不改变卡玛、夏普、收益率、最大回撤或三组对比的计算和视觉设计。

## 数据来源与关联

### 模拟持仓事实

富途模拟账户 API 是当前模拟持仓的唯一事实来源。每个市场使用已登记且彼此隔离的模拟账户：A 股、港股和美股不能交叉读取，也不能使用真实持仓或报告计划作为降级数据。

### 报告归因

通过成交执行事件中的市场、标的、方向、报告执行日期、报告哈希和策略版本，把当前模拟持仓关联到首次建立该仓位的已成交买入动作。只有存在确定的成交事实时才建立归因。

卖出后，该标的不再出现在当前模拟持仓，但报告文件、执行事件和历史入口永久保留。报告重新生成时保存新版本，不覆盖已经参与执行或已展示在历史中的版本。

### 持久化

继续使用现有文件存储：

- 冻结报告保存在现有市场报告目录；
- 动作与成交事实保存在现有不可变执行账本；
- Dashboard 只构建只读投影，不创建第二份持仓数据库。

历史报告按 payload 中的执行日期和显式版本排序，不能使用文件名日期猜测执行日期。

## 异常处理

- 富途模拟账户不可达或返回无效数据：显示“模拟盘暂不可用”及原因，不回退到真实持仓、报告计划或缓存示例。
- 单个模拟仓位无法关联报告：正常展示仓位并标记“未关联历史报告”。
- 单个历史报告损坏：该条目标记“报告不可读取”，其他历史报告继续可用。
- 同一仓位出现冲突归因：不选择任意一个版本，标记“报告关联冲突”并使验收失败。
- 新报告生成：只更新当前报告指针；不得删除旧报告、旧版本或旧执行结果。

## 验收条件

1. 每个趋势交易账户精确显示四个 Tab，顺序为“真实持仓｜模拟盘持仓｜趋势报告｜对应市场复盘”。
2. 默认激活真实持仓；切换 Tab 只替换当前账户内容。
3. 真实持仓现有字段、数量和交互不发生回归。
4. 模拟盘持仓代码、数量和成本与对应富途模拟账户 API 完全一致。
5. 可追溯仓位能打开精确报告执行日期与版本；无关联仓位仍显示并明确标记。
6. 新报告生成后，旧报告、旧版本和旧执行结果仍能通过趋势报告页内的历史入口查询。
7. 报告日期使用 payload 的执行日期；文件名不能作为执行日期权威。
8. 富途不可用时页面明确失败，不显示伪造或替代持仓。
9. 账户头部不新增卡片式入口、胶囊按钮或第五个账户级 Tab。
10. 375px 页面无横向滚动，Tab 可正常键盘操作且焦点清晰。
11. 自动测试、真实富途 API 检查、桌面与移动浏览器流程、进程与日志检查全部通过。
12. 最终 `make acceptance` 必须返回 `PASS`；`FAIL` 必须修复，`BLOCKED` 必须如实报告。
