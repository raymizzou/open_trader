# Dashboard 模拟盘与实盘对照设计

## 目标

美股趋势报告同时展示两个独立事实源，不再把老虎真实账户的持仓误解为富途模拟盘执行结果：

1. `模拟盘执行状态` 使用现有 `/api/trend-simulate-positions/<broker>` 返回的富途模拟账户持仓，与冻结报告逐项核对。
2. `实盘执行辅助` 保留现有老虎真实账户只读对照，明确它不影响模拟建议和自动执行。
3. Dashboard 中普通数值最多显示两位小数，不再暴露 Decimal 或浮点长尾。

这是纯展示修复，不改变策略版本、报告哈希、订单执行或账户数据。

## 最小实现

不增加 API、后台轮询或账户连接。前端在进入“趋势报告”视图时复用现有模拟盘持仓加载与缓存，并把同一份数据交给报告渲染函数。

报告风险区依次渲染：

- `模拟盘执行状态 · 富途`：主面板，默认展开。
- `实盘执行辅助 · <实际券商>`：次面板，继续使用现有 `actual_overlay`，默认按现有偏差规则展开或收起。

模拟盘核对按股票代码关联冻结报告动作与模拟持仓：

- `BUY` 尚无执行事件时显示“待执行”，不能显示成“跳过”。
- `HOLD` 且模拟数量大于零时显示“一致”；数量为零时显示“未持有”。
- `SELL_ALL` 且模拟数量为零时显示“一致”；仍有数量时显示“待卖出”。
- 报告外模拟持仓单独显示“报告外持仓”，不混入老虎实盘偏差。
- 模拟接口加载中或失败时显示明确状态，不回退到实盘数据，也不把未知解释为零持仓。

全局 `formatDisplayNumber` 继续负责千分位，但将有效数字限制为最多两位小数。整数不补 `.00`，例如 `485.0 → 485`、`30.594999999999995 → 30.59`、`23.428857… → 23.43`。

## 验收标准

自动测试和最终 `make acceptance` 使用相同的事实定义：

- 报告页同时可见“模拟盘执行状态”和“实盘执行辅助”，并标明富途模拟盘与实际券商两个来源。
- 模拟面板逐项使用模拟持仓 API；测试夹具中的 GPN `485`、TOST `1,296` 必须显示为“一致”，不能显示“未持有”。
- 实盘面板仍使用真实账户数据，且明确“只读对照，不影响模拟建议与自动执行”。
- 模拟接口不可用时必须显示不可用状态，不能复用实盘持仓。
- 桌面与移动浏览器流程都检查两个面板的数据源标签和核对结果。
- 两个面板和趋势报告内所有可见普通数值的小数部分不得超过两位。
- 最终仍按项目规则运行一次 `make acceptance`；只有 `PASS` 才部署和交付。

## 非目标

- 不合并模拟盘与实盘账户。
- 不新增持仓同步任务、缓存层或连接。
- 不改变买卖计划、数量计算、防重账本或下单流程。
