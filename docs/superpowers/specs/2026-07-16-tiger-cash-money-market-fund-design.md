# 老虎现金与货币基金 Dashboard 修正设计

## 目标

让老虎账户摘要按用户认知展示真实现金：货币市场基金折算为现金，普通基金仍作为持仓；可交易额度单列，不能混入现金。

## 已确认口径

- 同步老虎 `STK` 与 `FUND` 持仓，不再用 `TIGER_UNMAPPED_ASSETS` 代替可识别的基金。
- 名称明确为货币市场基金的 `FUND` 归类为 `money_market_fund`；其他基金归类为普通 `fund`。
- 老虎顶部“现金”是各币种净现金加货币市场基金市值。
- 货币市场基金不进入 SMA200 证券持仓表，也不计入证券持仓数量。
- 老虎的 `cash_available_for_trade` 单独显示为“可交易额度”，不计入现金。
- 老虎金额换算优先使用 Tiger OpenAPI 返回的实时外汇率；缺失或无效时保持现有数据校验行为，不猜测实时值。
- 现金与基金可以在账户摘要中展开查看来源，但不恢复已删除的全局现金 Tab。

## 数据流

1. Tiger 同步分别读取股票和基金持仓，以及账户资产与分币种现金。
2. 同步把真实基金写入现有 `Position`/明细/组合链路；货币市场基金使用已有 `AssetClass.MONEY_MARKET_FUND`。
3. 账户总资产调节只处理 API 仍未明细化的真实残差；已同步的货币基金不会再形成 `TIGER_UNMAPPED_ASSETS`。
4. Dashboard 的券商摘要把证券持仓与现金类资产分开汇总：现金明细和货币市场基金进入现金，其他 Position 进入持仓。
5. 老虎摘要额外展示可交易额度，并保持账户来源和刷新时间信息。

## 备选方案与取舍

- 继续保留 `TIGER_UNMAPPED_ASSETS`：改动最少，但无法说明资产是什么，且会继续污染持仓数；不采用。
- 把所有 `FUND` 都算现金：实现简单，但会误分类普通基金；不采用。
- 读取真实 `FUND` 并只把货币市场基金算现金：信息最准确，复用现有资产分类和组合链路；采用。

## 错误处理

- 股票查询成功但基金查询失败时，同步失败，不发布一个看似完整但漏掉大额资产的新快照。
- 无法识别的普通基金保留为 `fund`，不能静默算作现金。
- 实时汇率或必要金额无效时，不展示伪造的汇总金额，并沿用现有阻断/数据检查机制。

## 验证

- 单元测试先复现默认 `get_positions()` 只返回股票、遗漏 `FUND` 的问题。
- 同步测试证明货币基金以真实名称和市值写入，且不再生成对应的未明细资产调节项。
- Dashboard 测试证明货币基金进入现金、退出持仓金额与数量，普通基金仍留在持仓。
- Web 测试证明老虎摘要显示现金与独立的可交易额度。
- 运行 Tiger 真实只读同步并核对股票数、基金、净现金、现金合计与 API 时间。
- 最后运行 `make acceptance`；只有 `PASS` 才部署已验收 Git SHA 并提供 review URL。

## 非目标

- 不改变 SMA200 策略规则。
- 不恢复全局现金筛选或现金详情页。
- 不把购买力当作现金或可交易额度。
- 不修改其他券商的账户口径。
