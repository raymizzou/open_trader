# 东方财富 A 股结单与策略接入设计

## 目标

从用户提供的东方财富电子对账单中导入当前 A 股持仓，以 AKShare 补充日线行情，并让这些持仓使用 Open Trader 现有标准策略、回测和 Dashboard 展示。

## 范围

- 只解析对账单首页“汇总股票资料”和人民币现金数据。
- 只处理当前持仓，不读取资金流水或重建历史交易。
- 第一版只使用日线行情。
- 复用现有三种标准策略：趋势回调、突破动量、区间均值回归。
- 不接入东方财富实时持仓、EMT、EMQ 或自动下单。

## 数据流

1. 用户提供东方财富电子对账单 PDF 及本次导入所需密码。
2. 导入命令在内存中使用密码打开 PDF；密码不写入配置、产物或日志。
3. 东方财富解析器读取持仓代码、名称、数量、成本价、市价、证券市值，以及人民币资金余额和可用资金。
4. 解析结果沿用现有 `Position`、`CashBalance` 和 `portfolio.csv` 数据契约，市场统一为 `CN`，券商统一为 `eastmoney`。
5. 与其他结单来源相同，先写日期归档产物；只有显式更新时才覆盖 `data/latest/portfolio.csv`。
6. AKShare 为每个当前持仓调用 `stock_zh_a_hist` 获取前复权日线，转换为现有 `date,open,high,low,close,volume` 格式，保存到 `data/prices/CN/<symbol>.csv`。
7. 现有标准策略、回测和 Dashboard 从统一持仓与价格文件读取数据，不新增第二套策略链路。

## 行情选择

使用 AKShare 的东方财富 A 股日线接口，原因是它无需 API Key，直接提供现有策略所需的 OHLCV 字段，并支持沪深 A 股代码和指定日期区间。

第一版使用前复权数据，使近期价格与当前市场价格一致，同时保持技术指标所需的连续性。行情请求失败、返回空数据或字段不完整时，不覆盖已有价格缓存。

## Dashboard

A 股作为现有持仓看板中的普通市场展示，不创建独立页面或专属信息架构。

- 市场筛选新增 `A 股`。
- 券商筛选可显示 `东方财富`。
- 持仓分组新增 `A 股正股`。
- 持仓表继续使用现有列和顺序：明细、市场、标的、数量、成本价、实时价、美元市值、港元市值、持仓占总资产的占比、盈亏。
- 现有详情入口和响应式布局保持一致。
- A 股行情字段展示最近可用交易日收盘数据；页面不把结单导入描述成实时账户同步。

## 错误处理与安全

- PDF 密码只作为单次命令输入，不保存。
- PDF 无法解密、缺少持仓表或表头不匹配时，导入失败并保留现有持仓文件。
- 单个持仓行无效时报告具体行和字段，不静默生成零值。
- 行情失败时保留上次有效 CSV，并明确报告失败标的。
- 不在用户输出或日志中显示完整账户、股东账号、身份证号或 PDF 密码。

## 验收

1. 使用用户提供的真实东方财富 PDF 成功解析首页 5 只当前持仓和人民币现金，且不导入历史流水。
2. 日期归档产物与显式更新 `data/latest` 的行为符合现有结单工作流。
3. AKShare 为 5 只持仓获取真实前复权日线，生成现有价格 CSV 格式；价格缓存失败保护可运行验证。
4. 三种现有标准策略能读取 A 股日线并产出结果。
5. 相关自动化测试通过。
6. 本地 Dashboard 重启后，以真实 `data/latest` 验证 `A 股` 和 `东方财富` 筛选、`A 股正股` 分组，以及与其他市场相同的列顺序和行布局。
7. 用浏览器完成桌面和移动宽度检查，并核对新进程的 PID、启动时间及新日志。

## 明确不做

- 东方财富账户实时查询与持仓同步。
- EMT 或 EMQ SDK 常驻服务。
- 分钟级行情和盘中策略。
- 自动下单、撤单或订单状态管理。
- 通过历史资金流水推算持仓。
- A 股专属 Dashboard 页面或额外策略卡片。
