# 趋势报告与账户新鲜度解耦及 4% 回退仓位设计

## 目标

趋势报告只表达策略信号。账户快照是否为当天数据，不得删除、降级或阻止本来成立的买卖动作。Kelly 动态仓位接入前，三个市场统一使用 4% 单票仓位。

## 规则

- 美股、港股、A 股统一：策略动作仅由趋势、风险、流动性、持仓和保护线规则决定。
- 账户数据非实时也继续使用最近一次有效快照计算目标金额和建议股数。
- 非实时账户只显示提示：`账户数据非实时，执行前核对现金与持仓`。
- 不再显示 `已过期，禁止买入`，也不因 `account.fresh != true` 把买入动作转入人工复核。
- US、HK、CN 当前统一按账户净值的 4% 计算单票目标金额，不再使用 1% 试验仓位。
- 买入估算函数接收明确的 `position_weight`；日报入口在本阶段传入 `Decimal("0.04")`。
- 报告记录实际使用的仓位比例及来源 `fallback_4pct`，便于第二阶段替换为 Kelly 结果。
- 账户快照缺失或结构无效仍属于报告生成失败；本改动只处理“有效但非当天”的快照。
- 未知动作、未知原因、异常报告结构等既有人工复核和信任边界规则保持不变。

## 数据流

1. 报告生成器读取最近一次有效账户快照。
2. `build_report` 无条件按策略规则生成动作，并用该快照和 4% 回退仓位计算金额与数量。
3. 飞书和 Dashboard 展示相同动作。
4. 当 `account.fresh` 不为 `true` 时，仅追加执行前核对提示。

## 兼容性

- 不新增路由、依赖、配置或持久化结构。
- 不修改自动下单行为；报告仍是人工执行清单。
- 已冻结报告不原地修改。当天如需修正，生成 revision，且日账本继续阻止第二条飞书。
- 本阶段不收集成交样本、不计算 Kelly；US/HK/CN 独立真实成交样本及四分之一 Kelly 是下一份独立设计。

## 验证

- 报告生成测试：非实时账户仍产生合格买入动作，并按 4% 生成目标金额和建议数量。
- 飞书测试：保留买入动作，显示核对提示，不出现“禁止买入”。
- Dashboard 测试：买入计数和阶段内容不因 `fresh=false/missing/null` 消失。
- Acceptance 测试：原始冻结 JSON、API 投影和 DOM 三方动作一致。
- 全量测试及 `make acceptance` 必须 PASS。
