# Dashboard 暖灰账本与券商 Tab 设计

日期：2026-07-15

## 目标

统一 Open Trader Dashboard 及其所有内页的视觉语言，减少首页纵向长度，
让用户以券商账户为中心查看持仓与策略，同时保持现有数据、交易边界和
详情能力不变。

本次采用已确认的 B「暖灰账本」方向。该方向适用于首页以及从首页按钮
进入的策略回测、凯利实验室、趋势报告、交易决策、标的详情和研究对话。

## 首页结构

### 顶部导航

- 保留 `Open Trader` 与 `持仓实时看板`。
- 删除持仓区上方独立的凯利实验室入口条。
- 把 `策略回测` 与 `凯利实验室` 放在顶部，作为并列的同级次要入口。
- `刷新账户与行情` 是首页唯一主按钮。
- 市场筛选只保留 `全部市场 / US / HK / A 股`。
- 删除 `现金` 筛选以及独立现金详情页。

### 资产总览

- 最大资产卡固定显示 `全部账户总资产`，不随市场筛选或券商 Tab 改变。
- 持仓数量、当前券商资产和策略信息在券商 Tab 内展示，不与全局总资产混用。
- 四张券商汇总卡始终显示；点击卡片切换到对应券商 Tab。
- 券商汇总卡继续显示账户资产、交易周期、持仓数量和数据来源。
- 行情状态、四个来源状态和最后刷新时间继续位于首页顶部区域。

## 持仓与策略 Tab

- 只提供 `富途 / 老虎 / 辉立 / 东方财富` 四个券商 Tab，不提供“全部账户”。
- 页面首次加载默认选择富途。
- 复用现有券商筛选状态作为当前 Tab，不增加持久化存储。
- 刷新账户行情、打开或关闭行内详情后保持当前券商 Tab。
- 整页重新加载后恢复默认富途。
- 市场筛选在切换券商时保持不变。
- 市场筛选无匹配持仓时显示中文空状态，不自动切换市场或券商。
- 同步失败、数据缺失或零持仓的券商 Tab 仍然显示且可点击，并在内容区展示
  账户状态、中文错误原因和空状态。
- 现有持仓深链接继续可用；旧链接不含券商信息且同一标的存在于多个账户时，
  选择固定 Tab 顺序中第一个包含该标的的券商。

每个券商 Tab 依次展示：

1. 券商名称、账户别名、交易周期和数据状态；
2. 账户总资产、持仓资产、现金数值和持仓数量；
3. 账户策略摘要；
4. 当前市场筛选下的持仓。

现金只作为账户摘要数值显示，不再提供独立现金入口或币种明细页。

## 内页导航

- 策略回测、凯利实验室和趋势报告打开后隐藏完整资产总览，使用统一的紧凑顶栏。
- 紧凑顶栏显示 `Open Trader / 返回持仓 / 策略回测 / 凯利实验室`。
- 返回持仓后恢复原券商 Tab 和市场筛选。
- 交易决策、做 T、标的详情等现有行内展开行为不变。
- 研究对话继续使用模态界面，但采用同一颜色、按钮、输入框和层级规则。
- 不改变回测、凯利实验、趋势报告、研究对话或交易决策的数据契约。

## 视觉系统

### 基础色

| 用途 | 色值 |
| --- | --- |
| 页面背景 | `#FAFAF9` |
| 内容表面 | `#FFFFFF` |
| 主文字 / 深色操作 | `#1C1917` |
| 次要文字 | `#78716C` |
| 强调色 | `#A16207` |
| 边框 | `#D6D3D1` |

- 使用系统字体，不增加字体、组件或图标依赖。
- 使用清晰边框和克制阴影，不使用渐变或装饰动画。
- 数据列使用等宽数字并右对齐。
- 所有页面复用同一组语义色和组件状态，不为内页另建主题。

### 券商识别色

- 富途蓝、老虎橙、辉立绿、东方财富红。
- 券商色只用于细线、小圆点、Tab 指示器或极轻的选中态。
- 不使用大面积券商色背景。
- 券商名称始终显示，颜色不作为唯一识别方式。

### 盈亏与系统状态

- 盈利使用红色，亏损使用绿色，并始终保留正负号和数值。
- 系统正常或成功仍使用绿色，失败或风险仍使用红色，并始终显示文字标签。
- 交易盈亏和系统状态位于不同语义区域，不复用无标签的纯色提示。

## 数字格式

- 所有只读金额、价格、数量、统计值、回测结果和凯利实验数据使用英文千分位，
  例如 `3,064,187.62`、`2,932.00` 和 `10,000`。
- 保留数据原有的小数精度，不在格式化时重新计算或四舍五入。
- 百分比保留现有格式。
- 股票代码、账户号、日期、时间和其他标识符不使用千分位；例如 `02840`
  必须保持不变。
- 输入框继续使用原始数字文本，不加入逗号，也不修改输入解析。
- 格式化只发生在展示边界，不修改 API 数据或计算值。

## 响应式与可访问性

- 桌面端使用 B「暖灰账本」的横向顶栏、资产账页和下划线式券商 Tab。
- 手机端四个券商 Tab 保持单行等宽，不横向滚动、不吸顶。
- 手机端账户摘要纵向排列，持仓继续使用现有卡片结构。
- 页面和持仓区均不得产生横向滚动。
- 主交互目标不小于 44px。
- 保留键盘操作、清晰焦点、语义化 Tab 状态和现有 reduced-motion 支持。
- 正文文字达到 WCAG AA 对比度；状态不只依赖颜色。

## 实现边界

预期改动集中在：

- `src/open_trader/dashboard_static/index.html`
- `src/open_trader/dashboard_static/dashboard.css`
- `src/open_trader/dashboard_static/dashboard.js`
- Dashboard 对应的静态、单元和浏览器测试

复用现有券商筛选、账户分组、工作区切换和数字展示函数。不得新增前端框架、
组件库、字体、图标包、API、后端模型、策略计算或通知行为。

## 验证与验收

开发期间运行与改动对应的自动化测试和实际流程检查，不在每次修改后运行
`make acceptance`。

只有准备请用户验收 Dashboard 时，才按 `AGENTS.md` 运行 `make acceptance`。
验收必须覆盖：

1. 四个券商 Tab 的默认、切换、刷新保持和空状态；
2. 券商汇总卡切换对应 Tab；
3. 市场筛选跨券商保持；
4. 删除现金筛选与现金详情；
5. 全局总资产不随筛选或 Tab 改变；
6. 策略回测、凯利实验室和趋势报告的紧凑顶栏与返回状态；
7. 交易详情、研究对话和现有深链接；
8. 首页及所有内页的数字千分位与标识符例外；
9. 盈亏和系统状态颜色语义；
10. 桌面与手机布局、键盘焦点和无横向滚动。

只有 `make acceptance` 返回 `PASS`，才部署同一已验收 Git SHA，并在提供
review URL 前核对新 PID、工作目录、Git SHA、新日志和 HTTP 200。

## 不做

- 不增加“全部账户”Tab。
- 不保存券商 Tab 到本地存储或 URL。
- 不保留现金筛选或独立现金详情。
- 不增加暗色模式、图表、动画、排序、拖拽或自定义主题。
- 不改变交易、回测、策略、通知或账户同步逻辑。
