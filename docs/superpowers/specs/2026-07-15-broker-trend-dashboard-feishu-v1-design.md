# 券商趋势报告 Dashboard 与飞书通知设计 v1

## 状态

- 版本：v1
- 状态：已确认，可在后续版本迭代
- 日期：2026-07-15

## 目标

让每个已接入趋势交易的券商账户都能在 Dashboard 查看当天趋势报告，
并在报告生成后收到一条简短、可执行的纯文字飞书通知。

v1 覆盖：

- 富途：美股趋势报告。
- 辉立：港股趋势报告。
- 东方财富：A 股趋势报告。

## Dashboard 入口

每个券商账户板块的标题区显示一个 `当天趋势报告` 按钮。按钮旁必须同时显示：

- 报告日期：报告对应的执行日期。
- 数据截至：策略输入数据的日期。

进入报告后显示完整生成时间。报告日期、数据日期和生成时间不得合并成一个
含义模糊的“更新时间”。

当天报告存在时按钮可用；不存在时保留入口位置并显示 `今日暂无趋势报告`，
不得拿其他券商或其他日期的报告代替。

## Dashboard 报告布局

v1 采用已确认的 B 方案：`盘中时间线`。

页面顶部显示：

- 券商和市场。
- 报告日期、数据截至日期和生成时间。
- 账户数据状态。
- 卖出、买入、持有和人工复核数量。

主区域按执行顺序显示：

1. 开盘前：确认全部卖出动作及活动保护线。
2. 开盘窗口：显示允许买入的标的、数量、金额上限和预计保护线。
3. 盘中持续：显示继续持有及需要观察的保护线。
4. 人工复核：显示不能自动归类、必须人工判断的事项。

右侧在桌面端显示今日执行检查；移动端移到时间线下方。候选榜、排除项、
行业集中度、数据来源和 API 成本放入默认折叠的审计详情，不占据首屏。

买入时段固定为：

- 富途美股：`美股常规交易时段`。
- 辉立港股：`09:30–10:00`。
- 东方财富 A 股：`09:30–10:00`。

## 飞书发送规则

- 每个券商每个交易日生成一份报告，发送一条对应的飞书消息。
- 三个券商分别发送，不合并成一条总览。
- 当天没有买卖动作也发送，用于确认任务正常运行。
- 报告生成失败时发送该券商独立的失败提醒。
- 不发送报告更新或 revision 补发消息。
- 飞书发送的传输重试可以继续，直到同一条语义消息被明确接受；传输重试不算补发。
- 盘中保护线提醒属于 watcher，不受本设计的每日一条限制。

飞书消息只使用纯文字，不包含链接、按钮、卡片或 Markdown 表格。

## 飞书模板

### 有动作

```text
【{券商}｜{市场}趋势报告｜{报告日期}】

数据截至：{数据日期}
账户状态：{已更新 / 已过期，禁止买入}
今日动作：卖出 {数量}｜买入 {数量}｜持有 {数量}｜复核 {数量}

卖出
1. {代码} {名称}｜{中文原因}｜保护线 {价格}
2. {代码} {名称}｜{中文原因}｜保护线 {价格}

买入
1. {代码} {名称}｜{执行时段}｜约 {数量} 股｜金额上限 {金额}｜保护线 {价格}

人工复核
1. {代码} {名称}｜{需要确认的事项}

请人工确认，不自动下单。
```

规则：

- 完整列出所有买入和卖出标的，不截断。
- 持有只显示数量，不列出全部持有标的。
- 完整列出所有人工复核事项。
- 没有内容的 `卖出`、`买入` 或 `人工复核` 段落直接省略。

### 无买卖动作

```text
【{券商}｜{市场}趋势报告｜{报告日期}】

数据截至：{数据日期}
账户状态：{已更新 / 已过期}
今日无买卖动作｜持有 {数量}｜复核 {数量}

请人工确认，不自动下单。
```

### 生成失败

```text
【{券商}｜{市场}趋势报告生成失败｜{报告日期}】

原因：{中文失败原因}
现在做：{唯一、明确的恢复动作}

报告未生成，请勿依据旧报告交易。
```

生成失败提醒占用该券商当天的一条每日通知；恢复后可以手动生成报告，但不再发送
第二条飞书。

## 飞书不展示的内容

- Dashboard 地址或其他链接。
- 候选榜和排除项。
- 行业集中度。
- API 成本、缓存、内部字段或本地路径。
- 内部英文动作、原因或状态代码。
- 完整持有清单。

这些信息仍保留在 Dashboard 审计详情或 JSON 审计文件中。

## 数据流

既有趋势报告 JSON 是 Dashboard 和飞书格式化器的共同输入：

```text
富途美股报告 reports/trend_us_futu/*.json
辉立港股报告 reports/trend_hk_phillips/*.json
东方财富 A 股报告 reports/trend_a_share/*.json
                    │
                    ├── Dashboard：盘中时间线 + 折叠审计详情
                    └── 飞书：券商独立纯文字模板
```

Dashboard 和飞书不得重新计算趋势判断、仓位、候选排序或保护线。

## 异常与安全

- 缺少当天报告时，Dashboard 显示不可用状态，不回退到旧报告。
- 账户数据过期时，Dashboard 和飞书都必须明确显示 `禁止买入`。
- 未知动作或原因必须显示为人工复核，不能静默丢弃。
- 展示层失败不得修改报告 JSON、保护线状态或交易判断。
- 所有内容只读，不增加自动下单能力。

## 验收

最小验收覆盖：

- 三个券商账户板块都有当天趋势报告入口。
- 入口和报告页同时正确区分报告日期与数据日期。
- B 时间线在桌面和移动端均无横向溢出，执行顺序一致。
- 飞书有动作、无动作和失败三套模板均为纯文字。
- 所有买卖与人工复核项目完整展示，持有只显示数量。
- 飞书不包含 URL、按钮、卡片、审计附录或内部代码。
- 同一券商同一交易日只产生一条语义通知。
- 运行 `make acceptance` 并得到 `PASS` 后，才可交付 Dashboard 实现供评审。

## 非目标

- 修改富途、辉立或东方财富趋势报告的交易逻辑。
- 修改报告生成调度或市场日历。
- 修改盘中 watcher 的保护线通知。
- 引入飞书卡片、审批流或自动下单。
- 将多个券商合并成一条飞书通知。
