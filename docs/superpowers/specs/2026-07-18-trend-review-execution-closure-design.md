# 趋势复盘模拟执行闭环设计

## 目标

当趋势报告声明某个 `execution_date` 的正式动作时，系统必须在对应市场的有效交易时段执行富途模拟订单，并把报告、订单、成交、日终净值和复盘指标连成可追溯闭环。

本设计同时防止本次事故再次发生：美股 watcher 在修复提交前启动，长期保留旧代码；开盘回调仍按执行日匹配报告文件名，因报告实际按数据日命名而拒绝加载，导致 NDAQ 买入计划从未到达富途。

## 范围

覆盖 A 股、美股和港股三个趋势 watcher，以及：

- 报告修订选择；
- 富途模拟盘买入和卖出；
- 部分成交补单；
- watcher 重启恢复；
- 不可变执行事实；
- 日终纪律净值；
- Dashboard 执行状态；
- 通知与最终验收门禁。

不包含：

- 真实账户自动下单；
- 为验收制造额外测试订单；
- 手工操作专用模拟账户的兼容处理；
- 富途没有提供的逐笔模拟手续费估算；
- 新的 Dashboard 卡片或复盘结论。

## 时间与报告权威

### 执行日期

订单以报告的 `execution_date` 为准，不以报告生成自然日或文件名日期为准。例如，基于 7 月 16 日数据、7 月 17 日生成的美股报告，在其声明的 7 月 17 日美股常规时段执行。

报告文件继续按 `as_of_date` 命名。生产加载器必须接受 `2026-07-16[-rN].json` 中声明 `execution_date=2026-07-17` 的合法报告。

### 修订选择

执行前始终使用该执行日最新的有效数字修订。修订按标的生效，不整份冻结：

- 尚未生成 intent 的标的由最新修订接管；
- 标的一旦生成 intent，当日动作即冻结；
- 后续修订不得改变其数量、补仓或撤销已经提交的动作；
- 后续修订仍可新增、删除或调整其他尚未冻结的标的；
- 已成交但被最新修订删除的动作保留为“早期版本已执行”。

修订不能自动纠正已经成交的动作。报告修订本身不得触发反向交易；只有独立的正式卖出、活动保护线、危险信号或退出右侧趋势可以触发卖出。

## 动作身份与幂等

同日动作的稳定身份为：

```text
市场 + execution_date + 策略版本 + 标的 + 方向
```

报告哈希和动作序号只用于来源追溯，不作为幂等主键，因为报告修订会改变哈希和序号。

约束：

- 同一稳定动作只能有一个逻辑执行过程；
- 多次下单尝试属于同一动作，并有递增的尝试序号；
- 每次重试前先用稳定备注查询富途历史订单；
- 超时或连接中断时，只有确认券商不存在对应订单后才能重试；
- 同一标的同一执行日最多开启一个买入周期；当日清仓后不得因修订报告重新入场；
- 重试累计成交量不得超过目标数量。

## 执行闭环

### 触发方式

watcher 不得只依赖一次性开盘事件。以下时点都运行当日补偿扫描：

1. watcher 启动；
2. 市场进入正式交易时段；
3. 交易时段内每轮正常轮询；
4. 网络或 OpenD 故障恢复。

补偿扫描读取当天最新有效报告、不可变执行账本和富途历史订单，继续所有尚未达到终态的动作。

### 买入

- 只处理报告中正式 `BUY` 动作；
- 使用市价单；
- 报告没有声明价格失效条件时，不增加额外涨跌幅门槛；
- 目标数量使用模拟账户当时的净值、可用现金、策略目标权重、仓位上限和实时价格重新向下取整；
- 报告参考金额和参考价格只作解释，不覆盖模拟账户实时约束；
- 实时价格缺失、停牌、不可交易或计算数量为零时记录“受阻”，不猜测价格；
- 原订单仍有效时等待，不重复下单；
- 订单终止但累计成交不足时，按“目标数量减累计成交数量”继续提交市价单；
- 买入只在报告声明的有效窗口内补足；窗口结束后停止，不跨日补买。

A 股和港股现有买入窗口保持 09:30–10:00。美股使用报告声明的常规交易时段。晚到报告只要仍在有效窗口内即可立即执行；窗口结束后记录“错过”。

### 卖出

以下来源可以产生卖出：

- 报告正式 `SELL_ALL`；
- 活动保护线触发，包括跟踪止盈；
- 危险信号；
- 退出右侧趋势；
- 趋势温度转平。

卖出目标是模拟账户中该标的持仓归零。部分成交后继续按剩余持仓卖出；当天未完成则保留“待卖出”，以后每个可交易时段继续，直到富途持仓确认为零。卖出不受买入窗口结束限制。

正常生成器会用 `already_held` 排除已有持仓的买入候选。若坏报告仍对同一标的同时给出买入和卖出：

- 禁止买入；
- 模拟账户有持仓时保留 `SELL_ALL`；
- 模拟账户无持仓时忽略两者并记录冲突。

## 状态机

正常状态：

```text
待执行 -> 已提交 -> 部分成交 -> 全部成交
```

异常状态：

- `失败`：系统或券商明确返回错误；
- `受阻`：市场或必要输入暂不可用，窗口内可恢复；
- `错过`：有效窗口结束前从未成功提交；
- `未完成`：收市时仍有剩余数量；
- `早期版本已执行`：动作已经成交，但被后续报告修订删除。

网络中断和超时等临时错误在有效窗口内继续重试。明确拒单立即记录并通知；部分成交继续补足。买入窗口结束后不再重试新买单，卖出保持跨日待处理。

## 富途事实来源

真实 API 验证结果：

- 富途模拟账户不支持 `deal_list_query`；
- `history_order_list_query` 能返回订单状态、累计成交数量、成交均价、订单号和时间；
- 当前订单接口会漏掉已经完成的订单，不得作为日终复盘来源；
- 历史订单和账户接口都不返回逐笔模拟手续费；
- `accinfo_query.total_assets` 提供当前模拟账户组合净值；
- 富途没有提供模拟账户历史资产净值接口。

因此：

- 历史订单接口是模拟成交事实来源；
- `total_assets` 是纪律收益、夏普和卡玛的净值来源；
- 不将缺失手续费伪造为零，也不自行估算；
- 收市时必须冻结净值，失败后在下一次开市前持续重试；
- 历史订单可以补齐；跨过下一次开市仍未取得的历史净值不能精确恢复，只记录“净值缺失”，指标跳过该日并继续累计。

## 不可变复盘材料

不得覆盖提交时的 `SUBMITTING` 响应。每个状态变化追加一个不可变事实：

1. 动作来源；
2. intent；
3. 券商提交响应；
4. 后续历史订单对账事件；
5. 部分成交事件；
6. 全部成交、失败、错过或未完成终态；
7. 日终净值事实。

每笔纪律交易必须能够追溯到：

- 原始报告路径、修订号、报告哈希和动作序号；
- 稳定动作身份；
- 策略 ID、开仓版本和退出触发版本；
- 标的、方向、目标数量和每次补单数量；
- 富途订单号、订单状态、累计成交量、成交均价和时间；
- 触发原因、失败原因和通知事件；
- 对应交易日的模拟净值和市场基准。

部分成交合并为同一完整交易周期。只有从首次买入到最终持仓归零才增加一笔纪律样本。漏单只记录 `missed`，不增加样本；净值曲线继续使用模拟账户真实状态，不回填虚拟持仓或收益。

## 策略版本

策略版本只能在对应市场休市期间发布，并在下一交易时段生效。若仍有未清仓持仓：

- 新版本接管下一时段的后续判断；
- 活动保护线只能保持或上移；
- 完整交易归开仓版本；
- 退出事实同时记录触发退出的当前版本；
- 跨版本累计样本继续保留。

## 通知

- 明确拒单或系统错误立即通知；
- 临时等待或部分成交不立即轰炸；
- 在有效买入窗口结束与收市前 30 分钟两者中较早的时间仍未完成时通知一次；
- A 股和港股买入因此在 10:00 检查；
- 美股买入及风险卖出在收市前 30 分钟检查；
- 故障恢复后通知一次；
- 相同错误状态只通知一次，状态变化后才能再次通知。

## Dashboard

沿用已确认的 B 方案，在原买入或卖出交易行下增加执行明细，不增加新卡片、结论或操作按钮。

明细展示：

- 当前状态；
- 累计成交数量／目标数量；
- 成交均价；
- 富途订单号；
- 最近状态时间；
- 失败或受阻原因；
- 早期版本已执行时的来源修订。

桌面端保持现有表格。375px 手机端明细自然换行，不产生横向溢出、截断或隐藏信息。

## 验收条件

### 自动化验收

必须覆盖：

1. `2026-07-16.json` 报告可按 `execution_date=2026-07-17`执行；
2. 多个数字修订选择最新有效版本；
3. 已生成 intent 或已经成交的标的不因修订重复下单；
4. 被修订删除的已执行动作保留“早期版本已执行”；
5. 报告 `BUY` 产生正确市价买单；
6. 报告 `SELL_ALL` 产生正确清仓卖单；
7. 活动保护线、危险信号和退出右侧趋势产生正确卖单；
8. 同标的买卖冲突时禁止买入并按实际持仓决定是否卖出；
9. 部分成交只补剩余数量，累计成交不超过目标；
10. 买入窗口结束后不补买，未完成卖出跨日继续；
11. watcher 启动和故障恢复会补偿扫描，不依赖一次性开盘回调；
12. 富途超时先对账再重试；
13. 历史订单接口能恢复已完成订单，当前订单为空不能抹掉成交；
14. 提交、部分成交、全部成交、失败和错过事实均不可变且可追溯；
15. 日终净值缺失不伪造，也不阻止后续交易日继续累计；
16. 策略休市升级和未清仓持仓迁移符合版本规则；
17. 通知去重、窗口结束告警和故障恢复通知正确；
18. Dashboard 展示确认的 B 方案；
19. Dashboard 桌面端和 375px 手机端无横向溢出。

### 真实环境验收

对 A 股、美股和港股逐一检查：

1. 正常交易日存在一份最新有效报告；休市必须有交易日历证据；
2. 开盘前，未来动作显示“待执行”即可通过；
3. 进入有效窗口后，验收最多等待两个 watcher 轮询周期；报告中的每个应执行且当前可交易的动作必须出现唯一 intent 和富途订单；
4. 标的确实不可交易时必须有行情或券商返回的“受阻”证据；窗口结束后既无订单、又无受阻证据直接 `FAIL`；
5. 每个已提交动作都能在富途历史订单、执行账本和 Dashboard 中唯一匹配；
6. 富途累计成交量、成交均价、订单号和状态与本地不可变事实一致；
7. 收市后模拟净值、实际账户事实、真实成交事实和市场基准已经分别冻结；
8. 收市前 30 分钟仍未完成的动作已通知；明确失败已立即通知；
9. 不制造任何验收测试订单，只验证报告本来要求执行的动作；
10. `make acceptance` 最终返回 `PASS`，否则不得交付。

### 进程与部署验收

趋势执行相关代码通过最终门禁后：

1. 重启 A 股、美股和港股三个 watcher；
2. 验证旧 PID 已退出；
3. 验证新 PID、工作目录和 Git SHA；
4. 验证新日志包含 PID、启动时间和 SHA；
5. 验证三个 launchd 服务状态正常；
6. 验证启动补偿扫描已经运行；
7. 重新部署最终验收通过的同一 SHA，不改变源码或数据。

## 本次 NDAQ 漏单

上线后补偿扫描按统一规则处理：若 7 月 17 日美股常规时段仍未结束，则执行当前有效报告；若已经收市，只记录“系统错过”，不得在下一个交易日补买。
