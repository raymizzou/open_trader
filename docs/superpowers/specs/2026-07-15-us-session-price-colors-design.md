# 美股时段标签颜色设计

## 状态

- 日期：2026-07-15
- 状态：已确认
- 关联设计：`2026-07-15-us-session-price-dashboard-design.md`

## 目标

在不增加布局空间的前提下，让夜盘、盘前、盘中和盘后更容易快速区分。

## 已确认设计

只给现有时段文字标签着色；价格和行情时间继续使用现有文字颜色，避免把时段颜色
误解为涨跌或盈亏颜色。

| 时段 | 标签 | 颜色 |
| --- | --- | --- |
| 夜盘 | `夜盘` | 紫色 `#6941C6` |
| 盘前 | `盘前` | 橙色 `#B54708` |
| 盘中 | `盘中` | 蓝色 `#175CD3` |
| 盘后 | `盘后` | 青绿色 `#027A48` |

回退到上一有效价时，仍按真实来源时段着色。例如盘前没有报价而使用盘后价时，显示
青绿色的`盘后`标签和`上一有效价`。

## 实现边界

- 复用现有 `.session-quote-label`，为四个 `price_session` 增加稳定的语义标记和 CSS
  颜色规则。
- 不改变价格选择、估值、回退、时间格式、Header、watcher 或 API。
- 不增加胶囊背景、图标、图例、第二行或新的组件。
- 文本标签始终保留，颜色不是识别时段的唯一手段。
- 桌面端和移动端继续使用同一个紧凑单行布局。

## 验证

- 纯 JavaScript 测试覆盖四个时段的语义标记，防止标签与颜色映射错位。
- 静态样式测试确认四个语义标记各有唯一颜色规则。
- 现有 acceptance 继续验证每个美股价格单元格只有一个时段标签、移动端不溢出、
  Header 是唯一 CST 获取时间位置。
- 修改后运行 `make acceptance`；只有 `PASS` 才可交付。

## 非目标

- 给价格数值或行情时间着色。
- 用红绿表达涨跌。
- 允许用户自定义时段颜色或增加主题配置。
