# Dashboard 温润账本视觉校正设计

日期：2026-07-16

## 目标

把 Open Trader Dashboard 及其所有内页统一为已批准的 A「温润账本」视觉，
并修正趋势报告在桌面端明显窄于 Dashboard 的问题。

本次是纯视觉和布局校正。不得改变数据字段、文案、筛选逻辑、交易行为、
API 契约或后台流程。

## 当前问题

- 当前线上颜色是 `#FAFAF9 / #FFFFFF / #A16207`，并不等于批准的 A 色板。
- Dashboard 外壳最大宽度为 `1600px`，趋势报告另行限制为 `1180px`，导致
  桌面端左右留白过大，表格列过度换行。
- 15 列正式买入表即使扩大外壳，也不适合继续把全部内容硬塞进可视宽度。

## 精确颜色合同

以下颜色是实现和长期 Dashboard acceptance 的精确合同，不得用近似色替换：

| 语义 | 色值 |
| --- | --- |
| 页面背景 | `#F7F5F1` |
| 主表面 | `#FFFEFA` |
| 次级表面 | `#F2EEE7` |
| 主文字 | `#201D18` |
| 次要文字 | `#746E64` |
| 主交互与焦点 | `#8B5E34` |
| 边框 | `#D8D2C8` |
| 总资产深色卡 | `#24211D` |
| 风险与盈利红 | `#B42318` |
| 成功与亏损绿 | `#2F855A` |

这些前景与背景组合必须继续满足 WCAG AA 正文对比度。A 色板覆盖首页、
趋势报告、策略回测、凯利实验室、交易决策、标的详情和研究对话。

## 组件规则

- 主按钮、链接强调和键盘焦点环统一使用深棕主交互色。
- 次按钮使用主表面和暖灰边框。
- 总资产卡保留深色高对比表面。
- 富途蓝、老虎橙、辉立绿和东方财富红继续保留，但只用于 Tab 下划线、
  卡片左侧细线和小状态标记，不用于大面积背景或主操作。
- 盈利使用红色、亏损使用绿色；系统成功使用绿色、失败使用红色。所有状态
  都必须同时显示文字，不得只依赖颜色。
- 顶层面板使用克制阴影，内部卡片只使用边框，避免层层悬浮。
- 继续使用系统字体和 tabular 数字；不增加字体、图标、前端依赖或装饰动画。

## 布局规则

### 统一外壳

- Dashboard shell 继续使用 `max-width: 1600px`；趋势报告移除独立的较窄
  最大宽度并填满 shell 内容区。
- 桌面端 Dashboard 顶部、持仓区和趋势报告的左右边线必须对齐。
- 保留现有桌面与手机页面边距节奏，不增加新的布局容器抽象。

### 趋势报告宽表

- 桌面端普通趋势表优先使用可用宽度。
- 15 列正式买入表保留表格语义和横向比较能力，并设置足够的内容宽度；
  只允许表格容器内部横向滚动，字段不因硬塞而逐字换行。
- 页面根节点和趋势报告外壳不得出现横向滚动。
- `760px` 以下继续复用现有纵向卡片布局，每只股票一张卡，字段成对排列，
  不要求用户横向拖动表格。

## 交互与可访问性

- 不改变现有导航、返回、Tab、筛选、展开、刷新和报告入口行为。
- 主交互目标不小于 `44px`。
- 键盘焦点必须清晰可见。
- 不增加 hover-only 信息或布局位移式按压效果。
- 保留 `prefers-reduced-motion` 支持。
- 桌面和手机页面均不得产生页面级横向溢出。

## 实现边界

预期产品改动集中在：

- `src/open_trader/dashboard_static/dashboard.css`

长期 acceptance 合同的自动化保护集中在：

- `src/open_trader/dashboard_acceptance.py`
- `tests/test_dashboard_acceptance.py`

只有现有 DOM 无法满足已批准布局时，才允许修改 Dashboard HTML 或
JavaScript；出现这种情况必须先修订本设计，不得直接扩大实现范围。

## 验证与长期 Acceptance 合同

开发期间运行聚焦测试和直接浏览器检查，不在中间修改后运行
`make acceptance`。准备交付时，按 `AGENTS.md` 只运行最终 acceptance gate。

`make acceptance` 中现有 Playwright/Chrome 流程必须永久校验本设计，而不是
只检查 CSS 源文本。宽桌面 `1920px`、桌面 `1440px` 和手机 `375px` 至少验证：

1. 根元素精确 A 色板 CSS 变量；
2. 页面背景、主表面、次级表面、主按钮、总资产卡和代表性状态组件的
   `getComputedStyle`；
3. 在 `1920px` 视口中 Dashboard shell 为 `1600px` 最大宽度，趋势报告填满
   shell 内容区，并与 Dashboard 顶部及持仓区的实际左右边线一致；
4. 桌面宽表容器存在内部横向滚动能力，页面本身无横向溢出；
5. 手机趋势表变为卡片，页面本身无横向溢出；
6. 主要交互尺寸、可见焦点和现有真实导航流程；
7. 首页及全部内页继续使用同一 A 色板。

Acceptance 保存桌面和手机截图供人工复核，但不使用脆弱的逐像素截图差异
作为通过条件。精确颜色和几何规则由 Playwright 断言负责。

只有 `make acceptance` 返回 `PASS` 才能描述为完成。通过后必须按
`AGENTS.md` 部署同一已验收 Git SHA，核对新 PID、工作目录、Git SHA、新日志
和 review URL 的 HTTP 200，再向用户提供 URL。
