# 东方财富 A 股短线右侧趋势交易纪律

- 版本：v1
- 策略账户：东方财富 A 股账户
- 执行方式：系统生成操作清单，用户手工交易并手工登记成交
- 复盘周期：每完成 30 笔完整趋势交易

> 本纪律用于建立和验证个人趋势交易系统。趋势动物指标、操作清单与 Kelly 仓位均只作为决策参考，不构成投资建议；交易决定和盈亏由账户所有人承担。

## 一、核心原则

1. 只做趋势右侧，买强不买弱。
2. 判断错误时，用保护线截断亏损。
3. 趋势持续时不预设固定止盈目标，让利润奔跑。
4. 现金也是有效仓位，不为凑满持仓而交易。
5. 纪律给出建议，用户决定是否少买或放弃；不得放宽开仓门槛或突破仓位上限。
6. API 返回的是市场事实；买卖、仓位和风险处理是本纪律的策略判断，两者必须分开展示。

## 二、适用边界

### 2.1 账户

本纪律只管理指定的东方财富 A 股账户，不与其他券商账户或其他风格策略合并计算。

### 2.2 资产

- 纳入管理：账户内全部 A 股股票和 ETF。
- 首期市场范围：沪深股票及境内 ETF；排除北交所、ST、港股、美股、期货和可转债。
- 不纳入趋势仓位：现金、逆回购、货币基金。
- 无法识别或不受纪律支持的资产必须列为“账户例外”，不得静默忽略。

### 2.3 资金约束

- 不使用融资或其他杠杆。
- 不设最低持仓比例。
- 最多同时持有 10 笔未完成的趋势交易。
- 单票首次开仓不得超过账户策略净值的 4%。
- 10 个席位全部按 4% 开仓时，初始总仓位约为 40%；价格上涨后可以自然超过 40%，不因此减仓。
- 不设行业持仓上限。操作清单展示行业席位和仓位集中度，但不拦截主线行业候选。

## 三、数据来源与职责

| 数据源 | 直接事实 | 不负责的事项 |
|---|---|---|
| 趋势动物 API | 数据日期、候选成分、趋势右侧、强度、右侧天数、可交易性、成交额、危险及过热信号、行业 | 不计算本账户 Kelly 仓位，不替用户下单 |
| 富途行情 | A 股日 K、ATR14、最近 5 个完整交易日最低价、有效实时价格 | 不决定趋势动物信号 |
| 东方财富账户 | 策略净值、可用现金、实际持仓和实际成交 | 不自动改变纪律规则 |
| 本地策略 | 候选过滤、排序、固定目标仓位、保护线、操作清单 | 不把推断冒充 API 事实 |

### 3.1 趋势动物 API 调用纪律

1. 获取趋势、行情或截面数据前，先调用 `getUpdateStatus` 确认 A 股数据日期。
2. 定位品种或组合时，先用 `searchTicker` 获取 `tmId`，不得猜测内部代码。
3. 获取温转热组合成分时使用 `getComponentTicker`；文档要求时显式传入 `getAllBasicComponentsFlag`。
4. 调用 `getTickerSnapshot` 前先参考 `getSnapshotColumnBilling`，只请求本次任务所需字段。
5. `tmId` 查询结果本地缓存；付费响应按 `asOfDate + 接口 + 参数/字段集合` 缓存。同一数据日期不得重复购买相同数据。
6. 试运行阶段不设 API 日费用上限，但必须记录接口、字段、行数和实际费用。
7. 接口、参数、字段和计费规则始终以官方接口文档实时返回内容为准。文档未提供的信息必须写明“文档未提供”，不得猜测。
8. API Key 只放在被版本控制忽略的本地配置中，例如 `config/daily_premarket.env` 的 `TREND_ANIMALS_API_KEY`；文档、日志和仓库不得保存真实密钥。

### 3.2 最小快照字段

新开仓候选按任务请求以下事实：

- 标识：`tmId`、代码、名称、`asOfDate`
- 入口：`tradableFlag`、`amount1d`、`isTrendRightSide`、`daysSinceTrendEntry`、`trendStrengthLocalCurr`
- 风险：`stopwinFlagByDangerSignal`
- 展示：`industryName`（用于展示集中度，不作为行业上限）

已有持仓按任务请求：

- `isTrendRightSide`
- `stopwinFlagByDangerSignal`
- `stopwinFlagByBoilingTemperature`
- `stopwinFlagByPopChampagne`

## 四、纪律批次

1. 趋势动物确认 A 股数据更新后，每个 `asOfDate` 只生成一个正式纪律批次。
2. 该批次用于下一个交易日执行，不因看板刷新而重复生成。
3. 盘中只更新有效实时价格和保护线触发状态，不重算趋势动物日级信号。
4. 当日卖出动作优先于买入建议展示，但买入数量不得预支尚未成交的卖出回款。

## 五、新开仓纪律

### 5.1 唯一候选来源

新开仓只允许来自指定数据日期的趋势动物“温转热(A股)”与“温转热(ETF基金个股)”组合。两个组合的成分合并后使用同一套硬门槛、排序和持仓席位；“温转热(ETF基金)”返回分类节点，不作为可交易候选池。自选股、持仓股、AI 分析或临时热点都不能绕过候选池。

### 5.2 开仓硬门槛

候选必须同时满足：

1. `isTrendRightSide` 明确为 `true`。
2. 前一趋势温度为“温”，当前趋势温度为“热”或“沸”。
3. 筛选价格不高于 200 元。
4. `trendStrengthLocalCurr >= 95`。
5. 行业温度为“热”或“沸”。
6. 趋势节气为“谷雨”“立夏”或“夏至”。
7. 总市值不少于 100 亿元。
8. `amount1d >= 2`，即最近一日成交额不少于 2 亿元。
9. `daysSinceTrendEntry` 存在。
10. `tradableFlag` 明确表示可交易。
11. `stopwinFlagByDangerSignal` 未触发。
12. 交易代码不属于北交所，名称不含 `ST`、`*ST` 或退市标记。
13. 数据日期与报告日期一致，且不属于已有趋势持仓或待卖出持仓。
14. 能取得足够的日 K 数据并计算 ATR14。

任何必填字段缺失、为 `null` 或状态未知，都不得新开仓。

### 5.3 确定性排序

通过全部硬门槛、且不属于当前持仓的候选依次按以下顺序排列：

1. 本地趋势相对强度降序；
2. 右侧天数升序；
3. 最近一日成交额降序；
4. 股票代码升序。

排序后取前 10 只作为当日趋势候选；正式买入建议数量仍受剩余持仓席位约束。席位已满时，候选只保留在排序结果中，不自动顺延为未来买入。席位释放后，必须等待未来纪律批次重新确认。

前 10 候选用于展示当天最强的合格序列，不受账户席位和现金多少影响。只有生成正式买入动作时，才继续检查剩余席位、批次生成时已经可用的现金，以及目标金额能否至少买入 100 股；某只候选无法形成有效买入动作时，按上述确定性顺序考察下一只候选。

### 5.4 买入有效窗口

- 买入建议只在下一交易日 09:30–10:00 有效。
- 超过 10:00 未买入，记为“错过动作”，当日不得追买。
- 停牌、涨跌停无法成交或缺少有效实时价格时，记为“受阻动作”，不视为纪律违约。
- 同一持仓期间不补仓、不做 T、不因 Kelly 变化重新平衡。

## 六、仓位纪律

### 6.1 目标仓位

- 当前温度为“热”：目标金额为账户策略净值的 4%。
- 当前温度为“沸”：目标金额为账户策略净值的 2%。
- 当前运行版本不使用 Kelly 调整仓位。
- 价格上涨使单票仓位超过初始目标时不强制减仓；目标比例是首次开仓线，不是永久再平衡线。

### 6.2 建议金额与股数

- 目标金额以纪律批次生成时的账户策略净值和已经可用的现金计算，不计入计划卖出的预计回款。
- 报告按当天收盘价给出预估股数；09:30–10:00 内执行时，再按有效实时价格将目标金额向下取整为 100 股整数倍。
- 目标金额不足 100 股时建议跳过。
- 用户可以少买或放弃，但实际买入金额不得超过对应温度的目标仓位。
- 超出候选门槛、有效窗口或建议金额的成交标记为“纪律外交易”，不进入纪律模拟交易样本。

## 七、持有与卖出纪律

### 7.1 初始保护线

首次买入确认后立即计算：

`初始保护线 = 合并买入成交均价 - 2 × ATR14`

活动保护线初始等于初始保护线，此后只能上移，不能下调。

### 7.2 强制清仓

出现任一条件时，生成全部卖出动作：

1. `stopwinFlagByDangerSignal` 明确触发；
2. `isTrendRightSide` 明确变为 `false`；
3. 有效实时价格触及或跌破活动保护线。

卖出动作不设 10:00 截止时间，持续有效直至全部成交登记完成。未完成前持仓状态为“待卖出”，禁止重新买入。

### 7.3 让利润奔跑

`stopwinFlagByBoilingTemperature` 或 `stopwinFlagByPopChampagne` 首次触发时，不立即止盈，也不部分卖出，而是启用跟踪保护线：

1. 取此前 5 个完整交易日的最低价；
2. 活动保护线更新为原活动保护线与该最低价中的较高者；
3. 后续每个有效数据日期重新计算，活动保护线只升不降；
4. 有效实时价格触线时全部卖出。

本纪律不设“上涨 30% 卖出”、固定目标价或见到过热信号立即清仓。

### 7.4 再入场

全部卖出完成后不设固定冷静期。同一标的只有在未来纪律批次重新进入温转热候选池并通过全部硬门槛时，才能作为一笔新交易再次开仓；不得当日反手。

## 八、存量持仓

纪律启用时账户内已有的股票和 ETF 全部进入持仓检查集。

- 能取得完整买入成交与费用事实：按普通趋势交易管理。
- 无法取得完整事实：标记为“历史持仓”，不进入 Kelly 样本。
- 历史持仓以纪律启用时有效价格减 `2 × ATR14` 建立活动保护线。
- 危险信号、退出趋势右侧、过热跟踪和保护线触发规则照常适用。

## 九、人工执行与成交登记

### 9.1 操作清单不是订单

系统只生成建议，不连接东方财富自动下单。用户在东方财富手工执行。

### 9.2 必填成交事实

每次实际成交由用户手工填写：

1. 股票代码；
2. 买入或卖出；
3. 成交日期和时间；
4. 成交数量；
5. 成交均价；
6. 交易费用合计。

账户固定为东方财富，无需重复填写。系统据此计算成交金额和净收益。

同一操作在有效窗口内出现多笔部分成交时，按总数量、成交量加权均价和费用合计形成一笔合并成交。成交截图仅作为可选人工凭证，不替代手工登记。

## 十、异常处理

### 10.1 趋势动物未更新或不可用

- 不生成新买入，不沿用旧候选。
- 已有活动保护线和待卖出动作继续有效。
- 无法取得的 API 信号明确显示为“未知”，不得显示为安全。

### 10.2 单个持仓 API 字段未知

- 未知本身不触发卖出，也不代表安全。
- 保留原活动保护线并提示人工检查。
- 禁止该标的产生新买入。
- 只有 API 明确返回危险或退出右侧时才按该 API 信号强制清仓。

### 10.3 日线数据不足

- 无法计算 ATR14 或跟踪保护线时，不得新开仓。
- 已有持仓沿用上一条活动保护线，绝不下调。
- 趋势动物已经明确返回的强制清仓信号继续有效。

### 10.4 实时价格中断

- 停止计算建议股数。
- 不得把未知价格解释为保护线未触发。
- 立即提示“价格监控中断”，由用户在东方财富人工核价。
- 已有强制卖出动作继续有效。

### 10.5 富途服务整体不可用

- OpenD 断线、行情接口整体失败或交易日历不可用时，不生成正式交易计划。
- 从 17:00 起每 10 分钟重试，最晚到 18:00；仍失败则发送最终失败提示。
- 单个候选日 K 不足只排除该候选；单个持仓缺少新日 K时沿用已有保护线，不使整份报告失败。

## 十一、纪律版本与复盘

### 11.1 版本规则

- 运行中不得临时突破规则；调整只能通过新纪律版本生效。
- 每笔交易记录所用纪律版本。
- 纪律版本更新不自动重置 Kelly，样本继续累计。
- 只有用户明确宣布更换策略时，才建立新的 Kelly 样本池并重新进入 1% 冷启动。

### 11.2 每 30 笔复盘

从胜率、净盈亏比、平均持股周期、最大单笔亏损、持仓数量、错过动作和纪律外交易等角度复盘，并用以下四条标准检查下一版本：

1. **简单清晰**：选股、买入、卖出和仓位规则是否明确、可执行、互不冲突。
2. **截断亏损**：极端情况下是否存在无法接受的亏损，保护线和强制清仓是否真正有效。
3. **让利润奔跑**：肥尾行情出现时能否继续持有，是否存在过早止盈或人为限制主线暴露。
4. **符合个人习惯**：持有周期、执行频率、持仓数量、流动性和人工操作方式是否适合本人。

复盘只决定是否发布下一版本，不追溯修改已经完成的交易事实。

## 十二、A 股自动报告与盘中提醒

### 12.1 正式报告时间

- 每个 A 股交易日 17:00 生成以当天 `asOfDate` 命名、供下一交易日执行的唯一正式报告。
- A 股股票池、ETF 个股池和持仓快照的 `asOfDate` 必须一致且等于当天交易日。
- 数据未就绪时立即提示，此后每 10 分钟只检查更新状态，最晚到 18:00；仍未就绪则不生成正式计划，下一交易日不新买。
- 周末和 A 股休市日静默退出，不调用付费趋势接口。

### 12.2 账户新鲜度

- 用户每日维护东方财富持仓快照；当前确认数据作为初始基准。
- `portfolio.csv` 修改日期等于趋势数据日期时视为新鲜，否则标记账户过期。
- 账户过期时报告仍生成：展示前 10 候选，并对上次已知持仓给出条件计划；不计算正式买入金额或股数。
- 持仓超过 10 个时不因数量机械卖出，但暂停新开仓；所有持仓仍按趋势与保护线规则决定是否卖出。

### 12.3 本地与飞书产物

- 以 `asOfDate` 保存 Markdown 与 JSON；内容另行记录生成时间和计划执行日。
- 飞书展示全部持仓结论、前 10 候选、正式买入建议、数据状态和费用估算。
- JSON 保存 API 事实、过滤原因、纪律判断和数据来源；不得保存 API Key。
- 无买卖动作时仍生成正式报告，明确说明“现金也是有效仓位，本日无需交易”。
- 报告发送后冻结；只有用户显式要求时才生成并保留修订版。
- 飞书发送失败不撤销本地报告，也不重新调用付费接口；本地记录发送失败。

### 12.4 盘中保护线提醒

- 交易日 09:30–11:30、13:00–15:00 使用富途实时价格监控全部当前持仓的活动保护线。
- 首次触线时立即通过飞书和 macOS 提醒全部卖出，不自动下单；同一标的同一天只提醒一次。
- 次日持仓仍存在时，继续在正式报告中列为待卖出。
- OpenD 断线时立即提示监控中断，每 60 秒重连；恢复后提示监控恢复，断线期间不得声称保护线未触发。
